## 一、游戏概述

### 1.1 一句话描述

**怪物弹珠弹射操作 × 向僵尸开炮式三选一肉鸽构筑 × 山海归墟IP** 的竖屏小程序IAP弹珠游戏。

### 1.2 核心体验

玩家操控主角拉弹弹射，碰撞敌人造成伤害，碰撞随从触发技能连锁。通过三选一系统获得新随从、强化角色技能、获取全队增益，逐步打造独特build。在10波次持续涌入的战斗中击败精英和BOSS。

### 1.3 核心差异化

| 对比               | 向僵尸开炮            | 山海弹珠                                 |
| ------------------ | --------------------- | ---------------------------------------- |
| **操作**     | 自动战斗，无操作      | **手动拉弹弹射**，操作即策略       |
| **站位**     | 无意义                | **角色位置影响碰撞路径和技能触发** |
| **构筑**     | 三选一选技能          | 三选一选**随从+技能+增益**         |
| **乐趣来源** | 看build成型后自动碾压 | 操作碰撞路线 × build成型双重爽感        |

### 1.4 美术风格

沿用《山海归墟》的**沙雕熊猫人**美术风格：简洁、搞笑、辨识度极高。角色为Q版修仙沙雕造型，敌人为山海经异兽/妖怪。

### 1.5 目标用户

- 《山海归墟》现有粉丝群体
- 休闲/中度小程序游戏玩家
- 喜欢弹珠物理操作 + 肉鸽构筑的玩家

## 二、核心战斗系统（双模式设计）

> **设计说明**: 游戏支持两种战斗模式，通过分支版本进行对比测试，最终选择体验更好的模式作为主模式。

### 2.0 模式对比

| 特性                 | 回合制模式 (勇者斗恶龙式) | 即时制模式 (最终幻想ATB式)       |
| -------------------- | ------------------------- | -------------------------------- |
| **行动机制**   | 你一刀我一刀，轮流行动    | 时间流逝，行动槽累积，满即行动   |
| **敌人生成**   | 波次制，清场后下一波      | 实时持续涌入                     |
| **行动槽显示** | 无（回合提示）            | 角色周围环形描边                 |
| **节奏感受**   | 策略性强，节奏可控        | 压迫感强，紧张刺激               |
| **操作压力**   | 中等                      | 较高                             |
| **适合场景**   | 休闲玩家                  | 追求刺激的玩家                   |
| **内容一致性** | -                         | 与回合制完全一致，仅行动规则不同 |

## 二-A、即时制战斗系统（v2.1分支）

### 2A.1 核心机制：行动槽系统

#### 行动槽原理

```
每个角色拥有独立的行动槽（0-100）
    ↓
每帧根据速度值累加行动槽
    ↓
行动槽满（≥100）→ 角色可以行动
    ↓
行动后行动槽清零，重新累积
```

#### 速度值设计

| 角色类型   | 基础速度 | 说明                     |
| ---------- | -------- | ------------------------ |
| 主角       | 15       | 玩家操控，速度中等       |
| 反射型随从 | 18       | 敏捷型，行动频率高       |
| 贯通型随从 | 12       | 力量型，行动较慢但伤害高 |
| 远程型随从 | 10       | 策略型，行动最慢但射程远 |

**速度增益**: 通过三选一可获得速度+20%/+30%等增益

#### 行动槽UI（环形描边）

```
每个角色周围显示环形行动槽描边
┌─────────────────┐
│      ╭───╮      │
│     ╱ ███ ╲     │ ← 蓝色描边 = 主角
│    │  ◉◉  │     │
│     ╲ ███ ╱     │ ← 描边随行动槽累积
│      ╰───╯      │
└─────────────────┘

颜色区分：
- 主角：蓝色描边（半径40px，宽度5px）
- 随从：绿色描边（半径35px，宽度4px）
- 敌人：红色描边（半径40px，宽度5px）

行动槽满时：描边闪烁白色，提示可行动
```

### 2A.2 实时敌人涌入系统

#### 涌入机制

```
战场持续有敌人进入（非波次制）
    ↓
基础生成速率：每5秒生成1个敌人
    ↓
随着时间推移，生成速率逐渐加快
    ↓
场上敌人数量上限：8个（超过则暂停生成）
    ↓
击杀敌人获得分数，分数达到阈值触发精英/BOSS
```

#### 敌人生成曲线

| 游戏时间 | 生成间隔 | 敌人类型    | 压迫感 |
| -------- | -------- | ----------- | ------ |
| 0-30秒   | 5秒      | 只有山魈    | 低     |
| 30-60秒  | 4秒      | 山魈+毕方   | 中低   |
| 60-90秒  | 3秒      | 加入穷奇崽  | 中     |
| 90-120秒 | 2.5秒    | 加入盾甲兽  | 中高   |
| 120秒后  | 2秒      | 全类型+精英 | 高     |

#### 精英与BOSS触发

- **精英怪**: 击杀累计达到10个普通敌人后，下一个生成的是精英
- **BOSS**: 击杀精英后，延迟10秒生成BOSS
- **三选一**: 击败精英/BOSS后暂停游戏，弹出三选一

### 2A.3 双行动槽设计（敌我双方）

#### 敌人行动槽

```
每个敌人也有行动槽
    ↓
敌人速度固定（根据类型）
    ↓
行动槽满 → 敌人发动攻击
    ↓
攻击后行动槽清零
```

#### 敌人速度配置

| 敌人   | 速度 | 攻击频率       |
| ------ | ---- | -------------- |
| 山魈   | 8    | 较慢           |
| 毕方   | 10   | 中等           |
| 穷奇崽 | 15   | 快（高威胁）   |
| 盾甲兽 | 6    | 慢但高防       |
| 精英   | 12   | 中等但高伤     |
| BOSS   | 10   | 中等，但有技能 |

### 2A.4 战斗流程（即时制）

```
1. 战斗开始，所有角色行动槽开始累积
2. 敌人持续从屏幕边缘生成进入战场
3. 玩家角色行动槽满 → 玩家可以拉弹弹射
4. 随从行动槽满 → AI自动行动（可切换手动）
5. 敌人行动槽满 → 敌人发动攻击
6. 击杀敌人获得经验/分数
7. 击败精英/BOSS → 暂停战斗，三选一
8. 选择后继续，敌人继续涌入
9. 全队HP归零 → 游戏结束
```

### 2A.5 与回合制的核心差异

| 系统               | 回合制                      | 即时制             |
| ------------------ | --------------------------- | ------------------ |
| **行动顺序** | 固定循环：主→随1→随2→随3 | 速度决定，快者先动 |
| **敌人生成** | 波次清场后下一波            | 实时持续进入       |
| **敌人攻击** | 我方回合结束后统一倒计时    | 敌人独立行动槽     |
| **策略重点** | 单次碰撞路径规划            | 多角色行动时机管理 |
| **爽感来源** | 完美碰撞连锁                | 高频操作+持续清场  |
| **压力来源** | 敌人倒计时                  | 敌人不断涌入       |

### 2A.6 即时制特有机制

#### 1. 连击系统

```
短时间内连续击杀敌人 → 连击计数
连击数越高 → 伤害加成越高
连击中断条件：3秒内无击杀
```

#### 2. 紧急回避

```
当敌人行动槽满即将攻击时
玩家可以消耗SS能量进行紧急回避
回避：瞬间弹射到安全位置
```

#### 3. 战场压缩

```
随着时间推移，战场边界逐渐缩小
迫使玩家与敌人更密集交战
增加后期紧张感
```

## 二-B、回合制战斗系统（v2.0主版本）

### 2.1 弹射操作

| 要素               | 设计                                                         |
| ------------------ | ------------------------------------------------------------ |
| **操作方式** | 拉弹弹射（类似弹弓，拉角色本体松手弹出）                     |
| **瞄准辅助** | 显示**1段预测轨迹线**（到第1次碰撞为止），降低操作门槛 |
| **物理规则** | 角色弹出后碰撞墙壁/敌人按物理规则反弹，直到速度衰减停下      |
| **屏幕方向** | 竖屏，单手操作友好                                           |

### 2.2 角色类型（三分法）

| 类型                      | 碰敌行为               | 碰墙行为 | 自动模式行为               | 特点                               |
| ------------------------- | ---------------------- | -------- | -------------------------- | ---------------------------------- |
| **反射型 (Bounce)** | 反弹                   | 反弹     | 向最近敌人方向弹射         | 适合狭窄空间连续弹跳，多段碰撞伤害 |
| **贯通型 (Pierce)** | **穿透**（减速） | 反弹     | 向最近敌人方向弹射         | 适合一次穿过多个敌人，直线输出     |
| **远程型 (Ranged)** | 不弹射                 | 不弹射   | **原地释放**远程攻击 | 原地向敌人发射远程技能，站位策略型 |

### 2.3 主角与随从体系

#### 主角

- **固定参战**，始终在场上，始终由**玩家手动操控**弹射
- MVP固定为1个角色（后续版本可扩展多主角选择）
- 主角有自己的**技能树**，通过三选一强化
- 主角是**反射型**（最经典的弹珠操作体验）

#### 随从

- **局内通过三选一获得**，是肉鸽构筑的核心
- 场上**最多3个随从**（+ 主角 = 4个角色）
- 随从可切换**手动/自动模式**：
  - **手动模式**：轮到该随从时，玩家手动拉弹弹射（和主角相同操作）
  - **自动模式**：轮到该随从时，AI自动操作
    - 反射型/贯通型：自动向最近敌人方向弹射
    - 远程型：原地释放远程攻击技能
- 默认设为**自动模式**（加快小游戏节奏），玩家可随时切换为手动精确操控

### 2.4 伤害模式

- **碰撞伤害为主**：角色弹射过程中碰到敌人即造成伤害，碰撞速度/次数影响伤害
- 碰到**己方随从/主角**时，被碰到的角色释放其已有技能
- 远程型随从在自己的回合**原地释放技能**攻击敌人
- 纯正怪物弹珠风格：**角度与路径规划 = 核心操作技巧**

### 2.5 回合流程

```
1. 系统按顺序轮到下一个角色（主角→随从1→随从2→随从3→主角...）
2. 若为主角 → 玩家手动拉弹瞄准弹射
3. 若为随从(手动模式) → 玩家手动拉弹瞄准弹射
4. 若为随从(自动模式) → AI自动弹射/原地释放
5. 弹射角色碰撞敌人 → 造成碰撞伤害
6. 弹射角色碰撞队友 → 被碰队友释放其技能
7. 角色减速停下 / 远程角色释放完毕 → 回合结束
8. 全体敌人攻击倒计时 -1
9. 倒计时归零的敌人 → 发动攻击（对我方造成伤害）
10. 下一个角色进入行动阶段
```

#### 节奏优化（小游戏适配）

- 自动模式随从的弹射**加速播放**（不需要等完整物理模拟），约0.5秒快速完成
- 支持**快进按钮**（2x速度）
- 当场上只剩主角（无随从）时，节奏最快——纯单人弹射

### 2.6 敌人攻击机制

- **回合倒计时式**：每个敌人头上显示数字（如3、2、1）
- 每过一个**我方回合**，所有敌人倒计时 -1
- 倒计时归零 → 敌人发动攻击 → 倒计时重置
- **策略性**：优先击杀倒计时低的敌人，管理威胁优先级

## 三、肉鸽构筑系统（局内三选一）

### 3.1 设计理念

参考《向僵尸开炮》的核心构筑逻辑：**三选一是唯一的成长路径**。

每个角色（主角和随从）都有自己的**技能树**，通过三选一获取技能节点。随从本身也是构筑单元——获得什么随从、给谁升级什么技能，构成了每局独特的build。

### 3.2 三选一触发时机

| 触发条件                 | 说明                          |
| ------------------------ | ----------------------------- |
| **击败精英怪**     | 必定触发1次三选一             |
| **击败BOSS**       | 必定触发1次三选一（品质更高） |
| **波次清场里程碑** | 第2/4/6/8波清场后触发         |
| **特殊掉落**       | 普通怪有低概率掉落触发        |

> **「我全都要」功能**：三选一面板底部提供金色按钮「✨ 我全都要！」，点击后一次性获得当前三张卡的全部效果（适合局面压力大时快速补强）。

预计单局触发**6-8次**三选一，足够构筑出有体感的build。

### 3.3 三选一卡牌池

三选一从以下**统一卡牌池**中随机抽取3张，玩家选1张：

#### A. 招募随从（场上随从＜3时出现）

| 类型       | 示例                                  |
| ---------- | ------------------------------------- |
| 反射型随从 | 「剑尘心」反射/高攻击，被碰触发剑气斩 |
| 贯通型随从 | 「无敌问心」贯通/坦克，被碰触发震地波 |
| 远程型随从 | 「酒哈哈」远程/辅助，原地释放治疗弹   |
| ...        | 更多随从角色（MVP 8-10种）            |

每个随从自带**1个初始技能**，后续通过三选一升级/增加技能。

#### B. 角色技能升级（主角或已有随从）

| 类型     | 示例                                         |
| -------- | -------------------------------------------- |
| 攻击强化 | 「寒客·冰锥术」碰撞敌人时额外释放冰锥       |
| 范围技能 | 「剑尘心·万剑归宗」碰撞触发时释放全方位剑气 |
| 被碰强化 | 「酒哈哈·醉酒光环」被碰时释放范围回复       |
| 数值提升 | 「无敌问心·铁拳+1」碰撞伤害提升30%          |
| 新技能   | 给角色增加全新的技能效果                     |

#### C. 全队增益（随时出现）

| 类型     | 示例                          |
| -------- | ----------------------------- |
| 属性增益 | 全队攻击力+15% / 全队HP+200   |
| 特殊效果 | 碰撞产生冲击波 / 弹射速度+20% |
| 回复     | 立即回复全队HP 30%            |
| 被动光环 | 每回合自动回复HP 5            |

#### D. 随从替换（场上随从=3时替代招募类）

- 展示新随从，可以选择替换已有的1个随从
- 或选择不替换，改为获得全队增益

### 3.4 角色技能树结构

每个角色（主角/随从）拥有一个简单的技能树：

```
角色技能槽:
├── [主动技能] 弹射/行动时自动触发的技能（最多2个）
│   ├── 技能1: (初始自带 或 三选一获得)
│   └── 技能2: (三选一获得)
├── [被碰技能] 被队友碰撞时触发的技能（最多2个）
│   ├── 技能1: (初始自带 或 三选一获得)
│   └── 技能2: (三选一获得)
└── [被动属性] 数值加成（可叠加）
    ├── 攻击力+N%
    ├── 碰撞范围+N%
    └── ...
```

同一技能重复获得 → **升级**（最高3级），每级效果+30%。

### 3.5 构筑流派示例

| 流派                 | 核心思路                                                | 关键构筑                 |
| -------------------- | ------------------------------------------------------- | ------------------------ |
| **连锁爆发流** | 多个随从带AOE被碰技能，主角一次碰撞连锁多人引发多段爆炸 | 多反射随从 + AOE技能     |
| **远程炮台流** | 多个远程随从原地输出，主角负责碰触发增益                | 远程随从 × 3 + 主角增伤 |
| **贯通穿刺流** | 贯通随从一穿多，叠加穿透伤害和穿透次数                  | 贯通随从 + 穿透强化      |
| **速战速决流** | 全队攻击力拉满，快速清场                                | 攻击增益 + 高攻随从      |
| **永动续航流** | 叠回复+护盾，靠持久战消耗敌人                           | 远程奶妈 + 回复增益      |

## 四、波次设计（单局结构）

### 4.1 总体结构

- 单局共 **10波次**
- 敌人 **持续涌入**（场上始终有敌人，清完一批涌入下一批）
- 波次切换无暂停，紧凑节奏
- 三选一在特定节点**暂停战斗弹出**

### 4.2 波次详情

| 波次 | 类型       | 敌人配置            | 三选一触发                          |
| ---- | ---------- | ------------------- | ----------------------------------- |
| 1    | 普通       | 3只小怪             | —                                  |
| 2    | 普通       | 4只小怪（混合类型） | ✅ 清场后触发（引导性，必出随从卡） |
| 3    | ⭐精英     | 1精英 + 2小怪       | ✅ 击败精英后触发                   |
| 4    | 普通       | 5只小怪 + 引入陷阱  | ✅ 清场后触发                       |
| 5    | 普通       | 6只小怪（高密度）   | —                                  |
| 6    | ⭐精英     | 1精英 + 3小怪       | ✅ 击败精英后触发                   |
| 7    | 疯狂       | 8只小怪（持续涌入） | —                                  |
| 8    | 🔥BOSS     | BOSS + 2喽啰        | ✅ 击败BOSS后触发（高品质）         |
| 9    | 疯狂       | 2精英 + 大量小怪    | ✅ 击败精英后触发                   |
| 10   | 👑最终BOSS | 最终BOSS            | 通关！                              |

**单局三选一次数**：约 **6-7次**（波次2/3/4/6/8/9，加上可能的随机掉落）

### 4.3 持续涌入机制

- 场上同时最多存在 **25个敌人**（↑ 原6-8个，强化割草感）
- 开局直接生成 **10只**敌人（立刻有战场压迫感）
- 每回合补充速率：**wave1=4只/回合，wave10=9只/回合**（随波次递增）
- 补充敌人时使用「累加器机制」：每回合 `spawn_accumulator += rate`，累计≥1时生成1只
- 三选一触发时**暂停涌入**，选择完毕后继续

| 碰撞手感        | 弹射物理手感是核心体验，需大量调参    | Phase 1重点打磨        |
| 自动AI质量      | 自动弹射的AI效果直接影响体验          | 简单AI起步，逐步优化   |
| 三选一平衡      | 随从+技能+增益的强度平衡              | 持续playtest迭代       |
| 小程序性能      | 物理计算+特效的性能限制               | 预设性能预算           |
| Godot小程序导出 | Godot导出微信小程序的成熟度           | 提前验证，准备备选方案 |
| 远程型定位      | 远程型不弹射，可能感觉参与感低        | 通过技能特效增强存在感 |
